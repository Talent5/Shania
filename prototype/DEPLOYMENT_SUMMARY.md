## 🔧 Environment Variables

### Backend (Render)
```env
FLASK_ENV=production
FLASK_DEBUG=False
SECRET_KEY=[auto-generated by Render]
FRONTEND_URL=https://your-app.vercel.app
PYTHON_VERSION=3.9.0
```

### Frontend (Vercel)
```env
REACT_APP_API_URL=https://your-backend.onrender.com
```

## 📝 Deployment Checklist

### Prerequisites
- [ ] GitHub account created
- [ ] Code pushed to GitHub repository
- [ ] Render.com account created (free)
- [ ] Vercel.com account created (free)

### Backend Deployment (Render)
- [ ] Connect GitHub repo to Render
- [ ] Configure root directory: `backend`
- [ ] Set build/start commands
- [ ] Add environment variables
- [ ] Deploy and test health endpoint
- [ ] Copy backend URL

### Frontend Deployment (Vercel)
- [ ] Connect GitHub repo to Vercel
- [ ] Configure root directory: `frontend`
- [ ] Add `REACT_APP_API_URL` environment variable
- [ ] Deploy and test
- [ ] Copy frontend URL

### Final Configuration
- [ ] Update Render `FRONTEND_URL` with Vercel URL
- [ ] Wait for backend to redeploy
- [ ] Test full application flow
- [ ] Sign up, login, make prediction

## 🆘 Common Issues

**CORS Errors**: Make sure `FRONTEND_URL` is set correctly in Render
**502 Errors**: Backend is cold-starting (wait 30 seconds)
**Session Issues**: Both sites must use HTTPS (they do by default)
